Youtube Downloader/Splitter 10/17/16

Written in Java

There are many available Youtube downloaders out there that cost money or limit functionality. Moreso, I've always wanted a specific feature (which to the best of my knowledge simply isn't available on the market). My downloader uses multithreading to achieve fast download speeds. I've also developed a splitter functionality, which parses the description of the video for timestamps and converts/cuts/names files accordingly. This allows most albums to be downloaded and automatically cut into individual songs. This is very much a work in progress, but I hope to actively work on it for a while. The GUI isn't fully fleshed out, and functionality needs polishing. However, the program has hit the base functionality milestone. I'll soon be finishing many key aspects of the program and creating an installer.

Hotel Reservation Maker 2/12/16 - 2/19/16

Written in Java

Created for a potential employer, this project was both a learning experience and a showcase of what I can do in a week's time. It uses a local Access database to display available hotel rooms, taking into account existing reservations. It also stores user credentials for some basic login capability. Midway through the week-long trial, I took a big gamble overhauling the entire program to switch to an actual DB (okay, Access isn't a real professional database). It paid off, and I continued to try out a lot more new stuff (more sophisticated GUI layouts, prepared SQL statements, various Java 8 objects, dynamically changing listeners, etc.) for the first time. Probably the most progression I've had over the course of a single project. There is a lot that could be improved internally, but I'd rather spend time on applying the improvements on future code. Externally, the program is hard to break and performs as expected. It is officially closed. New users can be created, and the default user ajsnyde has the password "1234".

URL Fetcher 12/26/15

Written in Java

Another small side-project, created for giggles. It accepts a list of search queries and grabs the first youtube search result in URL form. It works by taking the HTML, using regex to find the result (filtering ads), and returning it in String form. This does NOT use the Youtube/Google API (this was intentional). Github repo is here. A newer multi-thread version has been released under minimal testing. This version has been optimized and can get roughly 600 translations per minute on my machine, roughly a tenfold increase over the other version. Obviously, over 1000 active http requests at once tends to break Youtube's built-in limit, so use with that in mind.
